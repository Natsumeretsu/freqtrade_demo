# 文档审查清单

更新日期：2026-01-17

## 1. 概述

**目的**：确保所有文档符合项目规范，保持高质量和一致性

**使用场景**：
- 创建新文档时
- 更新现有文档时
- 定期文档质量审查（每月第一周）

---

## 2. 基础检查项

### 2.1 元数据完整性

- [ ] 包含更新日期（格式：`更新日期：YYYY-MM-DD`）
- [ ] 日期格式正确（ISO 8601：YYYY-MM-DD）
- [ ] 如有版本号，格式正确（如：`版本：v1.0`）
- [ ] 如有状态标注，使用规范值（草稿/已完成/已归档）

### 2.2 文件命名规范

- [ ] 文件名遵循 `<主题>_<日期>.md` 格式
- [ ] 文件名使用小写字母和下划线
- [ ] 文件名清晰描述文档内容
- [ ] 日期格式正确（YYYY-MM-DD）

### 2.3 标题结构

- [ ] 只有一个 H1 标题（作为文档主标题）
- [ ] H2 用于主要章节划分
- [ ] H3-H4 用于细分内容
- [ ] 避免使用 H5、H6（不超过 4 层嵌套）
- [ ] 标题层级连续，不跳级

---

## 3. 内容质量检查

### 3.1 语言与风格

- [ ] 语言简洁明了，避免冗余
- [ ] 技术术语使用准确
- [ ] 中文标点符号使用正确
- [ ] 避免口语化表达
- [ ] 段落长度适中（3-5 句话，100-200 字）

### 3.2 代码块规范

- [ ] 所有代码块使用三个反引号（```）
- [ ] 代码块指定语言（如 ```python, ```bash）
- [ ] 代码块前后留空行
- [ ] 代码示例可运行或有明确说明
- [ ] 代码注释使用中文

### 3.3 列表规范

- [ ] 无序列表统一使用 `-`（不混用 `*` 或 `+`）
- [ ] 有序列表使用 `1.`, `2.`, `3.`
- [ ] 列表项之间不留空行（除非包含多段内容）
- [ ] 列表缩进正确（2 或 4 个空格）

### 3.4 表格规范

- [ ] 表格格式正确，对齐清晰
- [ ] 表头与内容分隔线正确
- [ ] 表格内容简洁，避免过长
- [ ] 复杂表格考虑拆分或使用列表

### 3.5 链接有效性

- [ ] 所有内部链接有效（指向存在的文件）
- [ ] 外部链接可访问
- [ ] 链接文本描述清晰
- [ ] 避免使用"点击这里"等模糊描述

---

## 4. 结构与组织

### 4.1 目录（TOC）

- [ ] 长文档（>1000 字）包含目录
- [ ] 目录放在引言之后、正文之前
- [ ] 目录深度适中（通常到 H3 级别）
- [ ] 短文档（<1000 字）可省略目录

### 4.2 章节组织

- [ ] 每个章节聚焦一个主题
- [ ] 章节长度适中（500-2000 字）
- [ ] 章节之间逻辑连贯
- [ ] 使用分隔线（`---`）分隔主要章节

### 4.3 引用与参考

- [ ] 引用来源标注清晰
- [ ] 参考资料列表完整
- [ ] 引用格式统一

---

## 5. 特定文档类型检查

### 5.1 技术报告

- [ ] 包含执行摘要
- [ ] 问题描述清晰
- [ ] 分析方法合理
- [ ] 结论有数据支撑
- [ ] 包含版本号和状态

### 5.2 评估报告

- [ ] 评估目标明确
- [ ] 评估指标清晰
- [ ] 数据来源可追溯
- [ ] 结果可视化（图表）
- [ ] 包含改进建议

### 5.3 知识文档

- [ ] 知识点提炼准确
- [ ] 包含实际应用场景
- [ ] 标注知识来源
- [ ] 定期更新检查

### 5.4 规范指南

- [ ] 规范条目清晰
- [ ] 包含正反示例
- [ ] 易于查阅和执行
- [ ] 长期稳定性考虑

---

## 6. 维护与更新

### 6.1 定期审查

- [ ] 文档内容仍然准确
- [ ] 技术信息未过时
- [ ] 链接仍然有效
- [ ] 示例代码可运行

### 6.2 归档判断

- [ ] 文档是否已过时（>6个月未更新）
- [ ] 内容是否被新文档替代
- [ ] 是否仅作历史参考
- [ ] 如需归档，是否添加重定向说明

---

## 7. 使用说明

### 7.1 审查流程

1. **创建新文档时**：
   - 完成初稿后，逐项检查本清单
   - 修正发现的问题
   - 标记所有检查项为已完成

2. **更新现有文档时**：
   - 重点检查修改部分
   - 更新元数据（日期、版本）
   - 检查相关链接是否需要更新

3. **定期审查时**（每月第一周）：
   - 批量检查文档质量
   - 识别需要归档的文档
   - 更新过时内容

### 7.2 审查工具

**自动化检查**：
- 使用 `scripts/tools/check_doc_metadata.py` 检查元数据
- 使用 MkDocs 构建检查链接有效性
- 使用 markdownlint 检查格式规范

**手动检查**：
- 内容质量和准确性
- 逻辑连贯性
- 可读性和易用性
