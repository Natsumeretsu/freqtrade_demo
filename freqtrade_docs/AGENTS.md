# freqtrade_docs 目录 - AGENTS 指南

本文件用于给 Codex/自动化助手提供 `freqtrade_docs/` 目录树范围内的工作约束与最佳实践。

## 1) 语言与风格（强制）

- 只使用简体中文编写文档正文（必要时保留关键英文术语/参数名/命令名）。
- 以“vibe coding 查阅/检索/复制”为目标：结构化、关键词友好、示例可直接复用；避免逐句翻译式搬运。

## 2) 文档产物约定

- `freqtrade_docs/raw_html/`：只存放离线保存的官方 HTML 原文（不做手改）。
- `freqtrade_docs/*.zh-CN.md`：整理后的中文 Markdown。
- 文件命名建议：使用主题小写 + 下划线，例如：
  - `data_download.zh-CN.md`
  - `configuration.zh-CN.md`

## 3) “vibe coding” Markdown 模板（推荐结构）

每篇文档建议包含：

1. 一级标题：中文标题 +（英文原名）
2. 2–3 行说明：说明是离线整理稿、偏快速上手
3. 元信息：
   - `来源`：官方页面链接（优先用 `rel=canonical`）
   - `离线保存时间`：从 HTML 顶部 `saved date:` 提取
4. `0)` 固定小节：本仓库推荐运行方式（Windows + uv）
5. 正文按“任务/问题”组织（编号小节 + 要点列表 + 最小示例）
6. 附录（推荐）：`原文（自动 Markdown 化）`，用于完整保留示例/表格/说明，便于全文搜索与复制。

代码块规范：

- 命令行用 `bash`（如有 PowerShell 差异可补 `powershell` 小节）
- 配置片段用 `json`
- 帮助输出/日志用 `text`

## 4) 内容取舍原则（避免又长又难读）

- 优先保留：
  - 常用参数/常见踩坑点
  - 参数之间的关系（例如 `stake_amount` 与最小下单额、`unfilledtimeout` 与限价单）
  - 关键的安全提醒（密钥、实盘、API 暴露等）
- 避免直接粘贴“超长参数表/完整枚举列表”；改为：
  - 给出常用子集 + 官方链接作为权威来源
  - 提供查询方式（例如 `freqtrade <命令> --help`、`show-config`）

## 5) 安全与脱敏（强制）

- 文档中禁止出现真实密钥/Token/密码/可用的 JWT secret。
- 示例统一用占位符：`<yourExchangeKey>`、`<telegramToken>` 等。
- 如需引用 `config.json`，应保证内容已脱敏（或用片段示例替代）。

## 6) 与本仓库运行方式对齐

本仓库使用 `uv` 管理环境，且仓库根目录就是 Freqtrade 的 `userdir`。

文档中出现官方 `freqtrade ...` 示例时，建议补一句说明：本仓库推荐写法为：

```bash
uv run freqtrade <命令> --userdir "." <参数...>
```

## 7) 校对清单（提交前自检）

- 是否包含 `来源` 与 `离线保存时间`
- 是否有 `0)` 小节说明本仓库运行方式
- 是否所有用户可见字符串均为简体中文（参数/命令除外）
- 是否有敏感信息泄露风险（密钥/Token/密码）
- 是否以要点和小示例为主，避免大段无结构文本
