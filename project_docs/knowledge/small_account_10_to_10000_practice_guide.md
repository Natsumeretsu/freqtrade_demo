## 执行摘要（先看这个）

1. 从 $10 增长到 $10,000+ 的关键不是“找到一个神策略”，而是构建一套可扩展的系统：**成本模型 + 风险预算 + 纪律化流程 + 可复现的回测/模拟/实盘验证**。
2. 小账户阶段（$10-$100）不是“赚钱阶段”，而是“**错误成本最低的训练阶段**”：你要训练的是下单与风控执行，而不是追逐收益曲线。
3. 如果你的目标包含“长期可持续”，那么你需要接受一个现实：**单靠复利把 $10 做到 $10,000 的难度极高**，更现实的加速器是“持续加本金 + 稳定正期望策略”，而不是“极端日收益”。
4. 工程化的底线公式：

\[
\text{净收益} = \text{名义收益} - (\text{手续费}+\text{滑点}+\text{资金费率/利息}+\text{机会成本}) - \text{尾部风险损失}
\]

## 1) 数学真相：复利的力量与陷阱

### 1.1 复利是对的，但“输入假设”经常是错的

复利公式很简单：

\[
A = P(1+r)^n
\]

- \(P\)：初始本金
- \(r\)：每期收益率
- \(n\)：期数

示例：$10，假设“每天 2%”的复利（仅用于展示数学，不代表可实现）：

| 天数 | 余额（$） | 倍数 |
|---:|---:|---:|
| 1 | 10.20 | 1.02x |
| 10 | 12.19 | 1.22x |
| 30 | 18.11 | 1.81x |
| 60 | 32.81 | 3.28x |
| 90 | 59.43 | 5.94x |
| 120 | 107.65 | 10.76x |
| 180 | 353.21 | 35.32x |
| 200 | 524.85 | 52.49x |

关键点：复利不是魔法，**它只会放大你输入的假设**；当假设不现实，复利曲线只是“幻觉放大器”。

### 1.2 “$10 → $10,000”到底需要多夸张的复合收益？

$10 → $10,000 等价于 **1000 倍**增长。若完全依赖复利，所需的复合收益随周期变化如下（用于校准预期）：

| 周期 | 需要的月复合收益 | 需要的日复合收益 |
|---:|---:|---:|
| 6 个月 | 216.23% | 约 3.91%（按 180 天） |
| 12 个月 | 77.83% | 约 1.91%（365 天口径） |
| 24 个月 | 33.35% | 约 0.95%（730 天口径） |
| 36 个月 | 21.15% | 约 0.63%（1095 天口径） |

结论：如果有人用“长期稳定”的语气谈论“每天 2%”，你应该立刻把讨论重心切回：**成本、回撤、连败、以及强平**。

### 1.3 回撤的非线性：你不是“亏 50% 再赚 50% 就回本”

从回撤 \(d\) 恢复到回本所需的涨幅：

\[
\text{恢复涨幅} = \frac{1}{1-d}-1
\]

| 最大回撤 | 回本所需涨幅 |
|---:|---:|
| -20% | +25% |
| -50% | +100% |
| -73% | +270% |
| -80% | +400% |

工程含义：小账户一旦出现大回撤，你会同时失去：
- 资金（可承受的误差更小）
- 机会（头寸变得更小，成本占比更高）
- 心态（更容易进入“复仇交易”）

## 2) 小账户最真实的约束（$10-$100 尤其明显）

### 2.1 最小下单与“成本地板”

小账户的问题不是“不会选币”，而是“**成本占比太高**”：
- 最小下单金额、最小合约张数、最小精度（stepSize）会迫使你用过大的仓位单位。
- 你的交易频率越高，手续费/滑点越接近“确定性亏损”。

一个实用的检查公式（以单次往返为单位）：

\[
\text{单次往返净优势} \approx \text{价差/波动捕获} - (2\times \text{手续费} + \text{滑点})
\]

当“单次往返净优势”长期接近 0 时，你会产生一种错觉：“我明明经常交易对了方向，但账户不涨。”

### 2.2 杠杆不是放大收益，是放大“尾部错误”

对小账户而言，“爆仓”通常不是来自连续小亏损，而是来自一次错误：
- 下单数量多敲一个 0（手数/合约单位错误）
- 忘设止损或止损类型用错
- 交易所波动穿透/跳价导致止损失效
- 保证金模式/仓位模式理解错误（逐仓/全仓、单向/双向）

工程化防线：把“事故类错误”当作一等公民来防，而不是只优化信号。

### 2.3 资金费率/借贷利息：对小账户是“慢性失血”

在永续合约中，即使价格不动，你也可能因为资金费率长期付费而亏损。小账户的关键不在于把资金费率算得多精细，而在于：
- 明确“是否允许隔夜持仓”
- 为“持仓跨结算周期”设定硬性规则
- 在成本模型里把资金费率视为“必然发生的摩擦”，而不是偶发事件

## 3) 三条可执行路径（按风险/时间投入分层）

> 说明：以下不是收益承诺，而是“可执行的工程路线图”。你应当用自己的回测与模拟交易去验证。

### 3.1 路径 A：稳健优先（训练 + 小额实盘 + 持续加本金）

适用人群：时间有限、希望长期坚持、愿意用加本金加速。

核心做法：
- 以“流程正确”为第一目标：回测 → 模拟 → 小额实盘。
- 目标从“翻倍”改为“**持续正期望 + 控制回撤**”。

阶段建议：
- $10-$100：优先模拟与训练，实盘只承担“学费”。
- $100-$1,000：开始形成稳定统计（交易笔数、胜率、盈亏比、最大回撤）。
- $1,000-$10,000：把风险比例调低，转向可扩展策略（低频/中频，避免高频成本吞噬）。

### 3.2 路径 B：中等激进（震荡类自动化 + 低频信号交易组合）

适用人群：愿意投入每周 5-10 小时做复盘与参数维护。

组合思路（示例）：
- 震荡段：网格/均值回归风格（但必须有“趋势段停机/切换”机制）。
- 趋势段：低频趋势/动量信号（减少交易次数，提升单笔期望）。

工程要点：
- 明确“体制过滤（regime filter）”：否则网格/均值回归会在单边行情中持续失血。
- 明确“交易频率上限”：否则手续费会吃掉所有边际优势。

### 3.3 路径 C：高风险激进（高杠杆/日内/短周期）

适用人群：把它当作“高风险训练项目”，并能接受高概率归零。

如果你仍要走这条路，工程底线是：
- 单笔风险必须极低（否则一次失误直接出局）。
- 必须有“当日止损上限/熔断”与“连续亏损停机”规则。
- 必须把“交易所机制风险（强平、穿仓、ADL、限额）”写入风险清单，并预留保证金缓冲。

## 4) 风险管理：从“口号”到“可执行规则”

### 4.1 两个硬规则：单笔风险与当日止损

在小账户阶段，比起精确的 Kelly 公式，更可用的是两条硬规则：

1. **单笔最大亏损**（含滑点）：通常建议 0.5%-2% 账户权益；越小越好，先保证能活过连败。
2. **当日最大亏损**：达到即停机（不允许“再来一单回本”）。

为什么？因为大多数爆仓来自行为失控，而不是统计学上的“正常波动”。

### 4.2 连败停机（反复验证有效的“反脆弱”机制）

示例规则（可按你的策略胜率/盈亏比调整）：

- 连续 3 次止损：暂停交易，复盘最近 3 笔是否“信号有效 + 执行合规”。
- 连续 5 次止损：当日停机；需要回测/模拟验证后才允许改参数。

### 4.3 头寸单位错误（最常见、最致命）

把它当作“高优先级事故”来处理：
- 下单前必须核对“名义价值（notional）”与“最大亏损（含杠杆）”。
- 如果下单确认页的名义价值超过账户权益的某个阈值（例如 80%），直接取消重算。

## 5) 行为与流程：为什么 95% 的小账户会失败

### 5.1 最常见的失败模式（按工程视角归因）

1. **过度交易**：把“频率”当成“努力”，实际等价于把手续费当成税。
2. **风险过大**：单笔风险 10%-30% 时，连败并不罕见，归零只是时间问题。
3. **复仇交易**：亏损后提高风险，只会把波动率变成“确定性死亡”。
4. **回测过拟合**：为历史极少数行情优化参数，实盘遇到体制切换直接崩溃。

### 5.2 复盘与日志：把“感觉”变成“可修复的缺陷”

最低可行的日志字段（建议固定模板）：
- 触发信号（具体条件）
- 计划止损/止盈（入场前就写好）
- 实际执行（是否滑点、是否漏单、是否手动干预）
- 结果（盈亏）
- 自评分（只评“是否遵守流程”，不评“赚没赚”）

## 6) 工程落地：Freqtrade + OKX（本仓库约定）

### 6.1 统一约束（避免踩仓库坑）

- 本仓库使用 `uv` 管理虚拟环境：`.venv/`。
- 所有 Freqtrade 命令通过脚本执行：`./scripts/ft.ps1 <freqtrade 子命令> ...`（会自动补 `--userdir "."`）。
- 禁止直接运行 `freqtrade` 或 `uv run freqtrade`，避免创建多余的 `user_data/` 子目录与路径混乱。

### 6.2 初始化与依赖

- 一键初始化：`./scripts/bootstrap.ps1`

### 6.3 获取历史数据（回测/训练前提）

示例（按需替换交易对/周期/范围）：

- `./scripts/data/download.ps1 -Pairs "BTC/USDT:USDT","ETH/USDT:USDT" -Timeframes "1h" -Timerange "20240101-" -TradingMode "futures"`

### 6.4 小账户相关的回测口径建议

小账户最容易被“回测幻觉”骗到的点：
- 回测默认不包含真实滑点与部分手续费结构（你需要在结论中明确“成本口径”）。
- 回测没有“下单失败/断线/订单状态不同步”等运维损失。

因此在写回测结论时，建议至少固定输出：
- 最大回撤（Max Drawdown）
- 交易次数与平均持仓时长（频率与成本敏感性）
- 费用敏感性：把手续费/滑点按 2 倍压力测试一次，看策略是否仍为正期望

### 6.5 网格策略的现实提醒（Freqtrade 不是交易所网格机器人）

网格在交易所侧通常是“多挂单、多层级、多头寸”的订单管理问题，而 Freqtrade 的常规策略范式更像“单笔交易的入场/出场”。因此：
- 如果你的目标是“交易所同款网格”，优先使用交易所自带网格产品（并把它当作外部执行器）。
- 如果你想在 Freqtrade 内实现“网格风格”，更现实的方式是做成“**均值回归 + 分批 + 体制过滤 + 频率上限**”的策略形态。

本仓库已有的“网格风格原型”（用于研究/回测口径校准）：`strategies/FreqaiVolatilityGridV1.py`

## 7) 里程碑：从 $10 到 $10,000 的“更现实”的衡量方式

把里程碑从“余额倍数”改为“过程质量”，你更容易跨过最难的阶段：

1. **第 0 阶段（可复现）**：同一份配置/策略，在不同月份的回测表现差异可解释，且不过度依赖单一行情。
2. **第 1 阶段（可执行）**：dry_run 运行 4-8 周无严重异常（漏单、重复下单、风控失效）。
3. **第 2 阶段（可控风险）**：实盘能做到“当日止损触发即停机”，且能坚持 3 个月不破规则。
4. **第 3 阶段（可扩展）**：账户增长后，风险比例下降但策略仍能保持正期望（容量/成本/体制适配都说得清）。

## 9) 可执行行动计划（7 天 / 30 天 / 90 天）

> 目标：把“想法”变成“可复现的过程”。如果你跳过前两段直接上实盘，小账户几乎一定会被成本与情绪击穿。

### 9.1 前 7 天：把环境与口径跑通（不追收益）

1. 初始化环境：`./scripts/bootstrap.ps1`
2. 选 1-2 个最主流币对（优先流动性）：例如 `BTC/USDT:USDT`、`ETH/USDT:USDT`
3. 下载历史数据（先别贪多）：`./scripts/data/download.ps1 -Pairs "BTC/USDT:USDT" -Timeframes "1h" -Timerange "20240101-" -TradingMode "futures"`
4. 选择一个现有策略/配置做基线回测（先验证“能跑通”而不是“跑得好”）
5. 固定输出口径：最大回撤、交易次数、平均持仓时长、手续费/滑点压力测试结论

验收标准（只要满足即可进入下一阶段）：
- 你能清楚解释“策略为什么赚钱/为什么亏钱”（而不是只报一个收益率）。
- 你能给出“当日止损上限”与“连败停机”规则，并能写进配置/流程。

### 9.2 前 30 天：dry_run 验证“执行链路”与“风控是否真的生效”

核心原则：dry_run 的首要目标是发现工程问题，而不是跑出漂亮收益曲线。

建议动作：
- dry_run 连续运行 2-4 周（期间只允许做“小改动”，不要频繁改参数）。
- 每周复盘一次：统计“触发信号 → 下单 → 成交 → 止损/止盈”的闭环是否一致。

验收标准（建议至少满足其中大部分）：
- 没有出现“重复下单/漏单/止损未触发/订单状态不同步”等事故类问题。
- 交易频率可控（你能解释每一笔交易为何发生，并能说出“为什么本周不该加频率”）。

### 9.3 前 90 天：小额实盘（把它当成“付费的风控训练”）

对 $10-$100 账户的现实建议：
- 目标不是“翻倍”，而是“**不破规则**”：只要不破规则，你就已经超过绝大多数新手。
- 单笔风险尽量用“绝对值”锁死（例如 $0.1-$0.3），避免百分比在小账户上失真。

硬规则（建议写进你的日常清单）：
- 达到当日止损上限：立刻停机（当天不许再开仓）。
- 发生任何“手数/合约单位错误/漏设止损”：当日停机并复盘（这类错误的再犯成本极高）。
